# 飞行棋大冒险发布说明

本项目有两个发布方向：网页版和微信小游戏版。两者共用项目文档，但入口、素材目录和测试方式不同。发布前先确认本次发布范围：只发网页版、只发微信小游戏版，还是两端同时发布。

## 发布入口

| 方向 | 目录 | 入口 | 发布平台 |
| --- | --- | --- | --- |
| 网页版 | `web/` | `web/index.html` | 本地浏览器、GitHub Pages、静态网站托管 |
| 微信小游戏版 | `minigame/` | `minigame/game.js`、`minigame/game.json` | 微信开发者工具、微信小游戏后台 |

根目录 `index.html` 是 GitHub Pages 兼容跳转页，会跳转到 `web/index.html`。不要把它当作主游戏源码维护。

## 网页版发布

本地预览推荐在仓库根目录启动静态服务：

```bash
python3 -m http.server 8765
```

然后检查：

```text
http://127.0.0.1:8765/
http://127.0.0.1:8765/web/index.html
http://127.0.0.1:8765/web/tests/test_planes.html
```

如果 GitHub Pages 使用仓库根目录发布，保留根目录 `index.html`，线上根地址会自动跳转到 `web/index.html`。如果托管平台支持指定发布目录，也可以直接把 `web/` 作为静态站点根目录。

网页版发布前至少检查：

- `web/index.html` 能打开。
- 根目录 `index.html` 能跳转到 `web/index.html`。
- `web/tests/test_planes.html` 能打开。
- `web/assets/audio/dice_roll.mp3` 能加载。
- `web/assets/images/plane_*.webp` 能显示。
- 首页 UI 图片、任务弹窗、骰子面板等素材能显示。
- 新游戏、快速开始、继续上局、掷骰、移动、任务弹窗等核心流程可用。
- 手机宽度和电脑宽度布局没有明显溢出。

## 微信小游戏版发布

微信开发者工具导入目录必须选择：

```text
minigame/
```

不要选择仓库根目录。小游戏项目关键文件为：

- `minigame/game.js`
- `minigame/game.json`
- `minigame/project.config.json`

当前小游戏代码主要使用这些相对路径：

- 小游戏专用素材：`assets/minigame/...`
- 骰子音效：`assets/audio/dice_roll.mp3`

这些路径都相对于 `minigame/`。移动素材时必须同步修改 `minigame/game.js`。

小游戏发布前至少检查：

- 微信开发者工具能打开 `minigame/`。
- 模拟器能启动游戏，不报入口文件错误。
- `minigame/project.config.json` 中 `compileType` 为 `minigame`。
- `minigame/game.js` 和 `minigame/game.json` 存在。
- 小游戏素材 `assets/minigame/...` 能加载。
- 音效 `assets/audio/dice_roll.mp3` 能加载。
- `minigame/project.private.config.json` 没有提交到 Git。
- 已在模拟器完成至少一次基础试玩。

## 双端同时发布

如果一次更新同时影响网页版和小游戏版，建议按以下顺序：

1. 确认更新范围：玩法、素材、文案、UI、存档或配置。
2. 先完成网页版修改并本地验证。
3. 再同步小游戏版对应逻辑或素材。
4. 分别完成网页版检查和小游戏检查。
5. 在 `CHANGELOG.md` 里明确写出两端影响范围。
6. 提交 Git，并按需要打版本 tag。
7. 先发布网页版，再发布小游戏版；或者先发布小游戏体验版，确认无误后再正式发布。

如果只更新其中一端，也要在更新记录里说明另一端是否需要同步：

```text
小程序同步说明：本次仅调整网页版发布结构，小游戏玩法无需同步。
```

## Git 发布流程

正式发布建议使用以下流程：

```bash
git status
git add ...
git commit -m "release: vX.Y.Z 发布标题"
git tag vX.Y.Z
git push origin main
git push origin vX.Y.Z
```

如果只是项目整理或文档调整，不一定需要打正式版本 tag；如果准备线上发布或对玩家可见，建议打 tag。

## 发布后检查

- 网页版线上地址能打开。
- 根地址跳转正常。
- 图片、音效没有 404。
- 浏览器控制台没有影响运行的报错。
- 微信小游戏开发版或体验版能进入游戏。
- 版本号、更新公告和实际内容一致。

## 常见问题

### GitHub Pages 打开后不是游戏页面

检查根目录 `index.html` 是否存在，并确认它跳转到 `./web/index.html`。

### 网页版图片或音效 404

检查资源是否位于 `web/assets/`，以及 `web/index.html` 中是否仍使用 `./assets/...` 路径。

### 微信开发者工具提示找不到入口

确认导入的是 `minigame/` 目录，并检查 `minigame/game.js`、`minigame/game.json` 是否存在。

### 小游戏素材不显示

检查 `minigame/game.js` 中的素材路径，当前应以 `assets/minigame/...` 或 `assets/audio/...` 开头。
