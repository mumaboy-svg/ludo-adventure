# 小游戏 v2.12.0：C4 设置系统同步

日期：2026-07-18
更新端：微信小游戏
对应网页参考版本：`v2.13.0`
小游戏版本：`v2.11.1 → v2.12.0`

## 版本评估

本轮增加完整设置分类、低动态偏好、数据状态与恢复默认流程，属于玩家可见的完整功能模块，因此小游戏功能版本升级为 `v2.12.0`。网页版保持已发布的 `v2.13.0`，两端独立管理版本号。

## 完成内容

- 设置页分为玩家配置、声音画面、数据管理三类。
- 玩家配置支持 2–16 人、改名、换色、删除、分页和独立保存。
- 短竖屏每页显示 3 位玩家；长竖屏与横屏每页显示 4 位玩家。
- 横屏使用双列玩家卡，适配 `844×390` 矮横屏。
- 设置页与通用页头根据左右安全区自动收缩，避让横屏刘海区域。
- 玩家名称按实际渲染宽度截断，避免宽字符与“改名”等按钮相碰。
- 背景音乐和低动态模式使用独立开关与持久化设置。
- 数据管理显示游戏存档、任务模式和当前任务数量。
- 恢复默认只重置玩家、背景音乐与低动态偏好。
- 没有存档时，“继续上局”显示无存档提示，不再自动创建快速游戏。
- 开始新局、快速开始、重开及任务保存后开局，在覆盖已有进度前统一确认。
- 修复已有存档冷启动时名称安全常量初始化顺序可能导致的启动异常。
- “继续上局”现在要求存档中存在有效玩家数组，损坏的非空对象不会再被误判为可继续存档。

## 数据边界

恢复默认只处理：

- `ludo_minigame_setup_v1`
- `ludo_minigame_bgm_enabled_v1`
- `ludo_minigame_reduced_motion_v1`

恢复默认明确保留：

- `ludo_minigame_state_v3`
- `ludo_minigame_tasks_v1`
- `ludo_minigame_task_packs_v1`

清空全部数据仍只删除以上六个明确存储键，不调用 `wx.clearStorageSync()`，不会误删其他小游戏或平台数据。

## 安全与兼容

- 保留玩家名称本地规范化与限制词过滤。
- 保留微信云函数 `msgSecCheck` 审核接入。
- 不修改 `minigame/project.config.json` 或 `minigame/cloudfunctions/`。
- 不修改骰子、棋盘、移动规则、任务触发、胜利判断或存档结构。
- 低动态模式只影响大厅、加载页和首页装饰动画，不影响游戏结果。
