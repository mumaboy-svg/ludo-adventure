## 双端共同契约

- 每次掷骰只创建一个冻结的 transaction。
- transaction 内只保存一个最终点数，范围为 1–6。
- 动画只展示 transaction 点数的派生帧，不得二次调用随机源。
- 动画结束后必须先显示 transaction 最终点数，再执行玩法结算。
- “最后点数”、最终骰面和玩法结算必须一致。
- 点数 2 为左上与右下；点数 5 为四角与中心。
- 动画期间必须锁定重复点击。
- 仅使用 `dice_roll.mp3`，音效失败不得影响游戏。
