# v2.15.0 网页版骰子动效同步说明

日期：2026-07-18

## 小程序同步包

【小程序同步包】

本次更新类型：优化 / 动效
网页版已改文件：

- `web/index.html`
- `web/tests/dice_c5_test.html`

小程序需要同步：可选，建议用户确认网页版效果后再做
如果确认同步，目标文件：

- `minigame/game.js`
- `minigame/tests/dice_c5_test.js`

同步内容：

1. 保留现有 Canvas 实体骰和 C5 唯一 transaction 结果契约。
2. 为骰子增加横向位移、第一次落地、第二次小弹跳和最终落定轨迹。
3. 阴影随骰子高度改变大小、透明度和模糊程度。
4. 可用轻量圆环或短促高光表现两次碰撞，不增加图片依赖。
5. 根据竖屏、矮横屏和骰子台尺寸缩小位移，避免越过容器。
6. 动画结束后先绘制 transaction 最终点数，再进入既有玩法结算。

关键规则：

- 六面与点阵继续使用 C5 已确认映射。
- 一次掷骰只生成一次最终随机结果。
- 动画不得调用第二次随机源。
- “上次点数”、最终骰面、任务回调和玩法结算必须一致。
- 重点验证 2 点和 5 点，不得出现空面或错误点数。
- 动画期间继续锁定重复点击。
- 音效继续只使用现有 `dice_roll.mp3`。

验证方法：

1. 运行小游戏骰子专项测试。
2. 连续掷骰至少 30 次，覆盖 1–6。
3. 单独固定测试 2 点和 5 点。
4. 检查最终点数、Canvas 画面和玩法结算一致。
5. 检查 `375×667`、`390×844`、`844×390` 的骰子运动不越界。
6. 检查低动态模式仍能快速、准确落定。

注意事项：

- 只改 `minigame/`，不要回改 `web/`。
- 修改前必须单独备份小游戏相关文件。
- 本轮网页版运动是程序曲线模拟，不是物理引擎。
- 小游戏同步属于独立更新，需重新评估小游戏版本号并建立独立更新说明。
- 不得顺带修改棋盘、玩法、任务、棋子移动、存档或胜利判断。
