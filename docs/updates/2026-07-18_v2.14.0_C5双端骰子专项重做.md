# v2.14.0 C5 双端骰子专项重做

日期：2026-07-18
版本：网页版 `v2.14.0` / 微信小游戏 `v2.14.0`
阶段：C5 骰子专项

## 本轮目标

停止继续修补旧骰子，将“随机结果、动画显示、最终结算”拆成清晰的三层，并确保 1–6 点稳定、2 点和 5 点准确、骰子像实体方块而不是翻牌。

## 实际修改

### 网页版

- 使用 CSS 3D 六面骰，增加实体厚度、阴影、跳动和落定反馈。
- 每次掷骰先创建唯一且冻结的 transaction。
- 动画帧只负责展示，最终骰面和玩法结算统一读取 transaction 的点数。
- 骰子本体与按钮都可点击，动画期间保留连点锁。
- 桌面骰子台使用固定安全列，避免与右侧按钮重叠。
- 仅在 `?dice-c5-test=1` 测试地址暴露最小测试钩子，正式页面不暴露内部函数。

### 微信小游戏

- 使用 Canvas 绘制正面、顶部和右侧三个可见面。
- 1–6 点共用统一点阵和三面朝向映射。
- 动画结束前先绘制最终点数，再解除 rolling 状态并进入结算。
- 骰子本体与操作区都保留触控入口和连点锁。

### 音效

- 双端只使用现有 `dice_roll.mp3`。
- 音效无法播放时静默降级，不影响随机结果或游戏结算。

## 结果契约

1. 一次掷骰只调用一次最终随机源。
2. transaction 点数创建后不可修改。
3. 动画过程中不得二次随机。
4. 最终骰面、最后点数和玩法结算必须使用同一 transaction 点数。
5. 点数 2 使用左上与右下；点数 5 使用四角与中心。
6. 动画未结束前拒绝重复掷骰。

- 网页实际掷骰：最后点数、最终属性、正面点数和圆点数量一致。
