# v2.20.3 小程序立方体骰子投影

## 版本评估

- 类型：修订版本。
- 原因：将小程序骰子从平面翻转视觉升级为六面立方体投影，不改变玩法或数据契约。

## 修改文件

- `minigame/game.js`
- `minigame/tests/dice_c5_test.js`
- `minigame/tests/motion_c6_test.js`
- `minigame/tests/story_guide_test.js`
- `CHANGELOG.md`

## 改动

- Canvas 骰子以六个面建立立方体顶点，按 X/Y 双轴旋转并做透视投影。
- 每帧按面向镜头的法线筛选可见面，再按深度从远到近绘制；正面、顶面和侧面会随翻转真实变化。
- 保留 820ms 常规动效、180ms 低动态模式、两次触地弹跳、动态阴影与原有触控边界。
- 六面骰点保留对面点数和为 7 的映射，最终点数仍由唯一 transaction 决定。

## 保留契约

- `createDiceRollTransaction()` 仍是骰子随机结果的唯一来源，`resolveDiceRoll(transaction.value)` 仍按原顺序结算。
- 未修改棋盘坐标、起飞/直道/终点规则、任务触发、国王卡、存档、首局引导或昵称审核。
