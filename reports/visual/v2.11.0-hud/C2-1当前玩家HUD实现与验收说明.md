# C2-1 当前玩家 HUD 实现与验收说明

## 实现范围

- 文件：`web/index.html`
- 玩家颜色飞机头像。
- 玩家名称、回合状态、位置阶段。
- 当前轮次、上次点数。
- 玩家切换动画与减少动态效果降级。
- 手机竖屏压缩布局和 `844×390` 矮横屏双栏布局。
- 横版大厅主按钮区域玻璃底移除。

## 明确排除

- 骰子随机与点数逻辑。
- 棋盘坐标和移动规则。
- 任务触发、任务库和存档结构。
- 微信小游戏 Canvas HUD 实现。

## 验收矩阵

| 检查项 | 预期 |
| --- | --- |
| `1366×768` | 棋盘与右侧 HUD 双栏共存，信息完整 |
| `844×390` | 保持双栏，不退化为上下单列 |
| `390×844` | 棋盘、HUD 和操作区无横向溢出 |
| `375×667` | 紧凑布局可用，文字不覆盖 |
| 玩家切换 | 头像、名称、状态与位置同步变化 |
| 掷骰结果 | 上次点数与骰子最终结果一致 |
| 控制台 | 无运行错误 |

## 验收结果

- `1366×768`：通过，棋盘与右侧 HUD 双栏共存。
- `844×390`：通过，布局保持双栏，HUD 文案完整。
- `390×844`：通过，无横向溢出。
- `375×667`：通过，紧凑布局可用；页面可按内容纵向滚动。
- 当前玩家飞机、名称、回合状态、位置阶段、轮次与上次点数均可见。
- 四个视口的骰子最终画面与 HUD 上次点数分别核对为 `1=1`、`6=6`、`3=3`、`2=2`。
- 网页内联脚本与小游戏脚本语法检查通过，`git diff --check` 通过。
- 唯一确认的 HTTP 缺失为浏览器自动请求的 `/favicon.ico`，不属于游戏功能资源。
