# v2.15.3 网页版骰子点比例优化

## 版本评估

网页版 `v2.15.2` -> `v2.15.3`。本轮是玩家可见的骰子样式微调，不增加功能、不改变玩法，因此升级修订版本号。

## 修改文件

- `web/index.html`
- `CHANGELOG.md`
- `docs/updates/2026-07-19_v2.15.3_网页版骰子点比例优化.md`

## 具体改动

- 骰子点直径由固定 `12px` 改为随骰子断点缩放的 CSS 变量。
- 桌面骰子点使用 `9px`。
- 普通竖屏骰子点使用 `8px`。
- 矮竖屏骰子点使用 `7px`。
- 加载页、页面标题和应用内可见版本号更新为 `v2.15.3`。

## 明确未修改

- 未修改 1-6 点位映射。
- 未修改随机 transaction、动画时序或最终点数。
- 未修改骰子本体尺寸、骰子台位置和掷骰按钮布局。
- 未修改棋盘、任务、棋子移动、存档或音效。
- 未修改 `minigame/`。

## 测试结果

- 网页内联 JavaScript 语法检查通过。
- 1-6 点位映射保持不变。
- 桌面、普通竖屏和矮竖屏均定义了对应骰点尺寸。
- 页面加载版本与 `APP_VERSION` 均为 `v2.15.3`。
- `390×844` 实测为 46px 骰子配 8px 圆点，无骰子区或页面横向溢出。
- `375×667` 实测为 40px 骰子配 7px 圆点，无骰子区或页面横向溢出。
- 两个手机视口的浏览器控制台均无错误或警告。
- 真实 QClaw 只读预检确认统一点选择器、1-6 映射和随机/动画逻辑不受本次尺寸调整影响。
- `git diff --check` 通过。

## 双端同步状态

本轮只调整网页版 DOM 骰子的 CSS 点直径。小游戏使用 Canvas 独立绘制骰子，不需要同步代码，继续保持 `v2.18.0`。
