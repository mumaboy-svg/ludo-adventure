# v2.8.2 双端信息页与小屏适配

**完成日期**：2026-07-17
**更新类型**：双端布局优化 / 小屏安全区修复
**涉及文件**：`web/index.html`、`minigame/game.js`、`CHANGELOG.md`、`docs/视觉美化规划.md`

## 更新背景

v2.8.0 将游戏进度和游戏记录拆成独立页面；v2.8.2 针对桌面横版、手机竖版和微信小游戏矮屏继续适配。重点解决返回按钮、内容列表、三列入口以及任务页底部按钮在有限空间中的显示问题。

## 网页版

- 信息页固定在当前视口内，卡片改为纵向弹性容器。
- 标题区保持固定，玩家列表和游戏日志使用剩余高度并在自身区域滚动。
- “返回游戏 / 返回大厅”改为等宽双列按钮，窄屏不挤压、不截字。
- 移除手机端 `calc(100svh - 184px)` 固定高度，避免不同分辨率下内容区失真。

## 微信小游戏

- 宽度低于 `400px` 时，任务中心、游戏进度、游戏记录三列入口缩小间距。
- 宽度低于 `96px` 的面板按钮使用 `13px` 字号，四字入口保持清晰。
- 信息页面板最低高度由 `320px` 调整为 `280px`，分页计算与实际渲染使用相同高度公式。
- 任务模式页和自定义任务页最低高度由 `430px` 调整为 `320px`，修复高度不足时底部操作按钮被裁切。
- 双端可见版本统一为 `v2.8.2`。

## QClaw 验收处理

QClaw 已只读检查网页与小游戏固定视口。采纳其“小游戏极矮屏任务模式页底部按钮被裁切”的确定性结论；报告中把 `safeBottom` 内边距误作屏幕坐标而得出的“统一溢出 56px”未采纳，并由 Codex 按实际坐标公式复核。
