# v2.9.2 小程序任务卡与国王卡排版修复

- 日期：2026-07-17
- 类型：微信小游戏弹窗排版修复
- 版本：Web `v2.9.2` / 微信小游戏 `v2.9.2`

## 更新内容

1. 国王卡和任务卡不再共用完全相同的固定文字坐标，分别按照实际 1:1 卡面设置标题、玩家信息、正文、按钮和关闭区域。
2. 玩家名称与任务来源恢复为独立的次级信息行。
3. 国王卡把“掷出 1 点触发”等原因与任务正文分层显示，避免两段文字挤成一段。
4. 正文换行函数保留文本中的显式换行；较长任务会逐级缩小字号，仍超出时使用省略号，避免越过卡面安全区。
5. 确认按钮上移到卡面底部边框以内，关闭按钮按两张卡面的顶部装饰分别定位。

## 未修改范围

- 卡面图片素材
- 任务文字与任务库
- 国王卡触发规则
- 棋盘规则和坐标
- 骰子逻辑
- 存档结构
- 网页版弹窗视觉

## 验证要求

1. 在微信开发者工具中分别触发普通任务卡和国王卡。
2. 检查短标题、长标题、两行正文和四行正文。
3. 检查国王卡的触发原因、玩家信息、正文彼此不重叠。
4. 检查确认按钮和关闭按钮位于卡面边框内且可点击。

## 小程序口同步说明

本轮小程序正式改动位于 `minigame/game.js`；卡面 PNG、任务库和触发规则未改。网页版仅同步可见版本号，保持双端发布版本一致。
