# C4 小程序设置系统同步任务单

【小程序同步包】

任务来源：Web `v2.13.0` C4 设置系统完整菜单化。

## 当前状态

- 网页版：`v2.13.0` 技术验收已通过，并已完成 Git 发布。
- 微信小游戏：继续保持 `v2.11.1`。
- Git：网页版 `v2.13.0` 已发布；本任务单不包含小游戏提交或平台发布。
- 本任务单不代表微信小游戏已经同步，也不提前修改小游戏版本号。

## 同步目标

在 `minigame/` 的 Canvas 设置页面中同步以下信息层级：

1. 玩家配置。
2. 声音与画面。
3. 数据管理。
4. 保存设置、恢复默认和清空全部数据的明确反馈。
5. 继续上局、返回大厅和开始新游戏的清晰入口。
6. 手机竖屏、矮屏和横屏安全区适配。

## 玩家配置要求

- 保留现有玩家数量范围。
- 保留玩家名称编辑。
- 保留玩家颜色选择和单色玩家上限。
- 显示当前玩家数量和游戏存档状态。
- 开始新游戏前明确提示是否会覆盖当前游戏进度。
- 玩家配置持久化方案必须先核对小游戏现有本地设置契约，不能直接照搬网页存储键。

## 声音与画面要求

- 背景音乐开关使用清晰的开启和关闭状态。
- 动态效果开关应提供可见反馈。
- 低动态模式只调整视觉动画，不影响骰子结果、棋子位置、任务触发和回合流程。
- 小程序端应使用 Canvas 原生控件和触控区域，不直接复制网页 DOM 尺寸。

## 数据管理要求

- 显示任务数据和游戏存档状态。
- “恢复默认”只重置玩家、声音和动态效果偏好。
- “恢复默认”不得删除游戏存档、任务库或任务模式设置。
- “清空全部数据”继续作为独立危险操作，并保留二次确认。
- 禁止使用无边界的全量清理替代既有数据管理流程。

## 必须保留

- 小游戏现有游戏存档键和存档结构。
- 小游戏现有任务数据、任务模式和任务内容。
- 小游戏现有音频资源和播放流程。
- 玩家数量、名称和颜色约束。
- 继续上局、返回大厅和开始游戏流程。

## 禁止修改

- 骰子随机、点数显示或骰子动画。
- 棋盘图片、坐标、起飞、直道、飞跃和终点规则。
- 任务内容、任务抽取和任务触发。
- 胜利判断。
- 现有游戏存档结构。
- 未经独立评估提前将小游戏版本改为 `2.13.0`。
- 未经用户确认执行 Git 暂存、提交、推送或微信平台发布。

## 小程序端验收要求

以下项目在同步实现完成后验证，本轮状态均为：**待主会话验收**。

1. 玩家配置分类可进入并正确操作。
2. 玩家增删、改名和颜色上限正确。
3. 声音与动态效果状态可保存并恢复。
4. 恢复默认不删除游戏存档和任务数据。
5. 清空全部数据保留确认流程。
6. 继续上局、返回大厅和开始新游戏入口正常。
7. 常见手机竖屏、矮屏和横屏下无按钮裁切。
8. Canvas 触控区域与视觉位置一致。
9. `node --check minigame/game.js` 通过。
10. 完成同步后再评估小游戏版本号、更新说明和发布流程。

## 同步边界说明

Web `v2.13.0` 的玩家偏好和视觉偏好实现仅作为产品与交互参考。小游戏同步前必须先读取小游戏现有存储键、设置数据和 Canvas 导航实现，按小游戏自身契约落地，不能直接复制网页本地存储代码。
