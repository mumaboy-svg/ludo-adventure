# C3 小程序任务中心同步任务单

【小程序同步包】

任务来源：Web `v2.12.0` C3 任务中心完整卡册化。

## 当前状态

- 网页版：已完成运行时验收，并随 Web `v2.13.0` 发布。
- 微信小游戏：保持 `v2.11.1`。
- 本任务单不代表小游戏已经同步，不提前修改小游戏版本号。

## 同步目标

在 `minigame/game.js` 的 Canvas 任务中心中同步以下信息层级：

1. 任务卡册标题和任务总数。
2. 家庭、聚会、情侣、轻松全集、手动任务五种模式入口。
3. 当前模式、玩家数量和存档状态。
4. 当前牌组摘要和开始游戏入口。
5. 手动模式的任务编辑、读取、保存、导出和检查入口。
6. 独立的清空全部数据危险操作区。

## 交互要求

- 预设模式隐藏手动任务编辑内容。
- 手动模式恢复任务编辑入口。
- 当前模式必须有清晰选中状态。
- 模式选择、保存和开局行为与网页版保持一致。
- 小屏高度不足时允许滚动或分页，底部按钮不能被裁切。
- 触控区域应满足手机操作，不照搬网页尺寸。

## 必须保留

- 五种模式及其现有逻辑映射。
- 任务字段、任务内容和任务库结构。
- 自定义任务、导入导出及本地存储契约。
- 玩家配置、存档读取和开始游戏流程。

## 禁止修改

- 骰子随机与点数显示。
- 棋盘坐标、起飞、直道、飞跃和终点规则。
- 任务触发和胜利判断。
- 存档键名或存档结构。
- 未经确认提前将小游戏版本改为 `2.12.0`。

## 验收要求

1. 检查五种模式均可选择。
2. 检查预设模式与手动模式的页面切换。
3. 检查开始游戏、返回、导入导出和清空确认。
4. 检查常见竖屏和矮屏尺寸，底部按钮不得裁切。
5. 运行 `node --check minigame/game.js`。
6. 完成同步并通过验收后，再评估小游戏版本号和发布说明。
