# v2.15.2 网页版竖屏骰子横向归位

日期：2026-07-19
版本评估：网页版 `v2.15.1` -> `v2.15.2`。本轮是玩家可见的竖屏布局纠正，使用修订版本号。

## 用户确认的正确布局

- 骰子不按整个屏幕单独居中。
- 骰子保持在自己的圆形骰子台中心。
- 骰子与“掷骰子”按钮位于彩色骰子面板的同一行。
- 骰子在左，按钮在右，按正常操作顺序排布。

## 修改文件

- `web/index.html`
- `CHANGELOG.md`
- `docs/updates/2026-07-19_v2.15.2_网页版竖屏骰子横向归位.md`

## 具体改动

- 普通竖屏将 `.dice-row` 改为 `72px + 自适应按钮` 两列，骰子台与按钮同排。
- 普通竖屏骰子台保持 `70px`，按钮高度调整为 `46px`，组合在彩色面板内容区垂直居中。
- 矮屏使用 `62px + 自适应按钮` 两列、`60px` 骰子台和 `34px` 按钮，减少整体高度。
- 保留 `.dice-container { overflow: visible; }`，避免实体方块旋转时被圆角底座裁切。
- 加载页和应用内版本号更新为 `v2.15.2`。

- 不修改棋盘坐标、玩法规则、任务、棋子移动、存档或音效。
- 不修改 `minigame/`。
