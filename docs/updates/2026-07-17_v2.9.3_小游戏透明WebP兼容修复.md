# v2.9.3 小游戏透明 WebP 兼容修复

- 日期：2026-07-17
- 类型：微信小游戏真机资源兼容修复
- 版本：Web `v2.9.3` / 微信小游戏 `v2.9.3`

## 问题

小游戏 `v2.9.2` 为降低上传包体，将 Logo、任务卡和国王卡转换为带透明通道的 WebP。开发者工具能够读取文件，但用户真机预览中三张图片均未显示。

## 更新内容

1. Logo、任务卡和国王卡恢复为标准 RGBA PNG，并将小游戏资源路径同步改回 `.png`。
2. 任务卡和国王卡从 1000×1000 缩放为 576×576，Logo 保持 640×640。
3. 保留三张图片的透明通道，避免影响现有卡面文字和背景叠加。
4. 双端可见版本号同步更新为 `v2.9.3`。

## 未修改范围

- 棋盘图片和棋盘坐标
- 游戏移动及直道规则
- 任务库和国王卡触发规则
- 存档结构

## 备份

修改前备份：`backups/2026-07-17_134100_before-v2.9.3-png-compat-fix/`

## 验证要求

1. 微信开发者工具重新编译并进行真机预览。
2. 检查大厅 Logo 是否显示。
3. 分别触发任务卡和国王卡，检查卡面图片、文字和确认按钮。
4. 确认上传包不超过微信小游戏 4MB 主包限制。
