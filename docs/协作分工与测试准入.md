# 飞行棋大冒险协作分工与测试准入

本文规定 Codex、Mavis 和 QClaw 在项目中的职责、权限和测试门槛。目标是尽可能分出低难度、可重复、结果明确的工作，同时避免辅助代理误改正式代码或核心玩法。

强制会话规则：实际启用代理前，必须分别建立独立的 Codex 专用任务，建议命名为“飞行棋大冒险 - Mavis 项目事务”和“飞行棋大冒险 - QClaw 自动测试”。不得在当前主任务中临时混用两个代理，也不得把两者放进同一个专用任务。专用任务只接收标准任务包，完成报告回到主任务后由 Codex 验收。

## 一、固定职责

### Codex

- 确定需求、视觉方向、交互方案和改动范围。
- 修改并验收正式代码。
- 处理骰子、棋盘规则、任务触发、存档等高风险模块。
- 评估版本号，决定回退、提交、合并和 GitHub 发布。
- 审核 Mavis/QClaw 的证据，不把“已执行”直接视为“已通过”。

### Mavis

适合处理文档和清单型事务：

- 素材盘点：文件名、尺寸、格式、WebP 状态、缺图项。
- 双端差异清单：页面入口、素材路径、可见文案和规则常量。
- 从真实 Git diff 草拟更新说明、同步说明和发布检查清单。
- 根据截图按客观项目列出遮挡、溢出、间距、对比度和缩放问题。
- 检查备份、版本说明、同步包和不应提交文件是否齐全。

### QClaw

适合处理可运行、可判定的技术测试：

- HTML/JavaScript 语法检查和浏览器控制台错误收集。
- 图片、音效、页面链接、文件名大小写和 WebP 引用检查。
- `APP_VERSION` / `GAME_VERSION` / CHANGELOG 的版本一致性检查。
- 固定桌面与手机视口的截图、横向溢出扫描和按钮可点击性回归。
- 按测试表重复执行大厅、任务中心、设置、开局、继续上局等低风险流程。
- 对 `web/` 与 `minigame/` 做只读差异报告。

## 二、任务风险等级

| 等级 | 范围 | 可执行者 |
| --- | --- | --- |
| L0 只读 | 盘点、搜索、差异、截图、报告 | Mavis、QClaw |
| L1 测试 | 运行已有命令和固定测试，不改文件 | QClaw；Mavis 可核对结果 |
| L2 隔离修复 | 仅在临时副本/独立分支做格式、路径、文档草稿等机械修复 | 通过准入后的 Mavis/QClaw |
| L3 正式低风险修改 | 正式源码中的 CSS、文案、素材引用、小范围 UI | Codex |
| L4 高风险 | 骰子、棋盘移动、规则、任务触发、存档、发布 | 仅 Codex，发布需用户确认 |

默认只授予 Mavis/QClaw `L0-L1`。即使任务很简单，也不得让它们直接修改当前正式工作区。

如果专用任务尚未建立，本轮由 Codex 自行完成检查，并在更新说明中记录“未调用代理”，不能假称已分配或已通过代理测试。

## 三、测试准入

首次扩大任务范围前，先给代理一个不改文件的试运行包。必须同时满足：

1. 正确识别项目根目录、网页入口 `web/index.html` 和小程序入口 `minigame/game.js`。
2. 完成网页脚本、小程序脚本和 `git diff --check` 检查。
3. 正确报告双端版本号，且不擅自修改版本。
4. 报告包含执行命令、退出码、失败证据和未检查项。
5. 明确列出没有修改的文件，并由 Codex复核 `git status`。

连续两次 L1 任务结果准确且未产生越权修改后，可授予单次 L2 隔离任务。L2 产物必须由 Codex 逐项审查后手工合入；一次误报、漏报或越权即恢复到 L0-L1。

## 四、禁止事项

Mavis 和 QClaw 都不得：

- 在正式工作区修改骰子、棋盘、任务触发、存档和规则。
- 自行决定视觉风格、版本号或是否发布。
- 执行 `git add`、`git commit`、`git push`、创建 PR 或发布 GitHub Pages。
- 删除文件、清理用户改动、覆盖未提交内容或使用破坏性 Git 命令。
- 把没有截图、退出码或差异证据的结论写成“测试通过”。

## 五、推荐任务分配

每轮默认分配：

- Mavis：修改前素材/文档盘点，修改后更新说明初稿与双端遗漏检查。
- QClaw：语法、路径、版本、响应式视口和固定功能回归。
- Codex：备份、正式修改、失败修复、最终验收和用户汇报。

图片生成提示词由 Mavis 根据 Codex 的美术规格整理初稿；用户确认素材后，格式转换、引用修改和视觉验收仍由 Codex 负责。

## 六、任务包模板

```text
代理：Mavis / QClaw
专用任务：任务名称或任务 ID
任务等级：L0 / L1 / L2（隔离）
目标：一句话描述可验证结果
输入：明确文件、截图、版本号或测试地址
允许操作：只读命令 / 测试命令 / 隔离副本修改
禁止操作：正式代码修改、Git 提交、发布、高风险模块
输出：结论、命令与退出码、证据路径、失败项、未检查项、未修改文件
验收人：Codex
```

## 七、首批可立即分配任务

### Mavis

1. 全量素材清单与 WebP 状态报告。
2. 网页/小程序 UI 与素材差异清单。
3. 每轮独立更新说明初稿和同步说明完整性检查。
4. Git 状态分类：本轮文件、历史未提交文件、禁止发布文件。

### QClaw

1. 建立语法、版本号、路径和 `git diff --check` 的基础测试报告。
2. 建立 390×844、430×932、1366×768、1920×1080 四个视口的截图回归。
3. 检查横向滚动、文字溢出、按钮可见性和控制台错误。
4. 建立大厅到新游戏/快速开始/继续上局/任务中心/设置的冒烟测试。

骰子 1-6、棋盘入口与反弹规则可以由 QClaw 在测试环境执行测试，但测试用例和预期值必须先由 Codex 锁定，QClaw 不能依据测试结果自行改规则。
